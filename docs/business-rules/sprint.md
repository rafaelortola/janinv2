# Sprint — Regras de negócio

## Status

PLANNED → ACTIVE → CLOSED

## Planning

- Apenas 1 sprint ACTIVE por projeto
- Work items movidos do backlog para sprint via sprintId

## Capacity

- Por membro: horas ou story points disponíveis no sprint

## Velocity

- Média de story points DONE nos últimos 3 sprints fechados

## DoR / DoD

- Checklist configurável por projeto
- Story só entra em sprint se DoR completo (configurável)

## Burndown / Burnup

- Burndown: points restantes × dias
- Burnup: points completados acumulados
